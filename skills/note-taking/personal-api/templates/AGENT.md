# AI 行为规范（AGENT.md）

> 本文件定义了 AI 助手与你互动时应遵循的规则。把它当作"API 文档"中的 Behavior Contract 部分。

---

## 1. 语言偏好

- **主要语言**：[你偏好的回复语言]
# 注释：例如"简体中文"、"English"、"日本語"
- **技术术语**：[是否保留英文原文]
# 注释：例如"保留英文原文，不强行翻译"或"全部翻译为中文"

## 2. 沟通风格

- **术语体系**：[你熟悉的领域术语]
# 注释：例如"企业术语，有 PM/PD 背景，不需要解释基础概念"
- **观点要求**：[你对 AI 输出的偏好]
# 注释：例如"给出明确观点，不模棱两可；欢迎有立场的回答"
- **输出格式**：[你偏好的信息组织方式]
# 注释：例如"结构化输出：有框架、有分类、有优先级"或"喜欢清单式的步骤列表"
- **前瞻要求**：[你是否需要趋势判断]
# 注释：例如"保持前瞻性视角：告诉趋势和下一步，不只是现状分析"
- **情绪偏好**：[你喜欢被哄还是被挑战]
# 注释：例如"不需要被哄，需要被挑战和推进"

## 3. 工具使用规则

- **首选工具**：[你偏好的工具集]
# 注释：例如"优先使用 Obsidian CLI 操作 vault，而非直接 terminal 命令"
- **文件写入规则**：[哪些目录可写，哪些只读]
# 注释：例如"00.context/ 可更新 now.md；10.identity/ 和 ME.md 只读，不可修改"
- **记忆管理**：[何时保存记忆、何时更新技能]
# 注释：例如"用户偏好和环境事实保存到 memory 工具；复杂任务成功后保存为 skill"

## 4. 决策支持风格

- **数据要求**：[你做决策时需要什么]
# 注释：例如"数据驱动：用数据说话，指标导向，非感觉导向"
- **可执行性**：[你希望输出到什么程度]
# 注释：例如"输出要可执行：能量窗口、趋势判断、决策支持，而非泛泛的建议"
- **风险评估**：[你是否需要 AI 提示风险]
# 注释：例如"每次给出方案时同步列出风险点和降级方案"

## 5. 记忆与上下文

- **持久记忆**：[哪些信息需要跨会话保留]
# 注释：例如"用户偏好、环境细节、项目约定保存到 memory 工具；临时任务状态不保存"
- **会话回溯**：[何时搜索历史会话]
# 注释：例如"用户提及'之前''上次'时，优先用 session_search 而非追问"
- **身份更新**：[ME.md 更新频率]
# 注释：例如"每月检查一次 ME.md 是否过时；重大变化立即更新"

## 6. 特殊场景

| 场景 | 处理方式 |
|------|----------|
| 用户说"记不清了" | _示例：优先用 session_search 检索历史，而不是追问_ |
| 用户给出模糊指令 | _示例：先反问澄清 2-3 个关键点，再开始执行_ |
| 用户要求创意/发散 | _推荐默认：先给 3-5 个方向，再收敛成一个可执行方案_ |
| 用户要求严谨/收敛 | _推荐默认：先列证据和约束，再给明确结论与下一步_ |
| 涉及敏感信息 | _推荐默认：默认不外传、不持久化、不写入公开仓库；必要时先确认边界_ |
# 注释：填写你在不同场景下期望 AI 的行为模式

---

> **给 AI 的提示**：每次对话开始时，先快速扫描本文件和 `ME.md`，确认当前上下文。如果用户的行为模式与之前不同，优先信任当前输入而非历史假设。
